## 一、本课目标

学完本课，你应该能做到：

1. 理解聊天模型返回的不是普通字符串，而是 `AIMessage` 对象。
2. 分清 `content`、`text`、`content_blocks`、`tool_calls`、`usage_metadata`、`response_metadata` 的用途。
3. 掌握把模型输出转成普通字符串的基础方式。
4. 掌握 `StrOutputParser` 的作用和用法。
5. 认识自由文本输出的局限，为下一课结构化输出做准备。

前两课我们重点学习“如何把问题交给模型”。这一课开始学习“模型回答之后，程序如何接住结果”。

## 二、为什么要专门学习模型输出

很多初学者第一次调用模型时，会以为模型返回的是字符串：

```python
response = model.invoke("什么是 LangChain？")
print(response)
```

但在 LangChain 中，聊天模型返回的通常是一个 `AIMessage`。

这不是多此一举。真实 LLM 应用里，模型输出不只是一段文字，还可能包含：

- 文本回答。
- 工具调用请求。
- 多模态内容块。
- token 用量。
- 模型名称、结束原因等响应元数据。
- provider 原始返回中的扩展信息。

如果只把输出当字符串，就会丢掉这些重要信息。

你可以先这样理解：

```text
普通字符串：只适合展示给人看。
AIMessage：既能展示给人看，也能被程序继续处理。
```

---

## 三、AIMessage 是什么

`AIMessage` 表示模型生成的一条回复消息。

在 LangChain 1.x 官方 Messages 文档中，AI 消息常见属性包括：

| 属性 | 含义 | 初学阶段怎么理解 |
| --- | --- | --- |
| `text` | 文本内容 | 适合直接获取纯文本 |
| `content` | 原始内容 | 可能是字符串，也可能是内容块列表 |
| `content_blocks` | 标准化内容块 | 统一表示文本、推理、多模态、引用等内容 |
| `tool_calls` | 工具调用 | Agent 和 Tools 阶段重点学习 |
| `usage_metadata` | token 用量 | 用于成本、性能和调试 |
| `response_metadata` | 模型响应元数据 | 例如模型名、结束原因、provider 信息 |
| `id` | 消息唯一标识 | 方便追踪和调试 |

第一阶段先重点掌握：

- `text`
- `content`
- `usage_metadata`
- `response_metadata`

`tool_calls`、`content_blocks` 后面在 Tools、Agent、多模态和 LangGraph 中会继续展开。

## 四、示例 1：查看模型返回对象

新建文件：

```text
examples/lesson_03_inspect_ai_message.py
```

代码：

```python
from os import getenv

from dotenv import load_dotenv
from langchain_openai import ChatOpenAI


load_dotenv()


model = ChatOpenAI(
    model=getenv("OPENAI_MODEL", "gpt-5-nano"),
    temperature=0.2,
)

response = model.invoke("请用两句话解释什么是 LangChain。")

print("对象类型：")
print(type(response))

print("\ntext：")
print(response.text)

print("\ncontent：")
print(response.content)

print("\nusage_metadata：")
print(response.usage_metadata)

print("\nresponse_metadata：")
print(response.response_metadata)
```

运行：

```powershell
uv run examples/lesson_03_inspect_ai_message.py
```

观察重点：

- `type(response)` 应该是类似 `AIMessage` 的消息对象。
- `response.text` 更适合直接拿纯文本。
- `response.content` 是原始内容，很多纯文本场景下也是字符串。
- `usage_metadata` 可能包含输入、输出、总 token 数。
- `response_metadata` 可能包含模型、结束原因、provider 返回信息。

注意：不同模型供应商返回的元数据字段可能不同，所以不要写死依赖某个 provider 的私有字段。

## 五、`text` 和 `content` 的区别

初学阶段容易把 `text` 和 `content` 混在一起。

可以先这样记：

| 属性 | 更适合什么时候用 |
| --- | --- |
| `response.text` | 你只需要纯文本，想展示、保存、传给下游字符串处理 |
| `response.content` | 你想查看原始内容，或需要保留 provider 返回的内容结构 |
| `response.content_blocks` | 你要处理标准化的多块内容，例如推理、引用、多模态、工具相关内容 |

在很多简单文本模型调用中：

```python
response.text
```

和：

```python
response.content
```

看起来可能差不多。

但从教材长期路线看，更重要的是理解：

> `AIMessage` 是完整输出对象，文本只是其中一部分。

后面学习 Tools 时，模型可能不是直接回答文本，而是返回 `tool_calls`。后面学习多模态或推理内容时，也可能需要检查 `content_blocks`。

## 六、示例 2：把输出保存为普通字符串

新建文件：

```text
examples/lesson_03_text_output.py
```

代码：

```python
from os import getenv

from dotenv import load_dotenv
from langchain_openai import ChatOpenAI


load_dotenv()


model = ChatOpenAI(
    model=getenv("OPENAI_MODEL", "gpt-5-nano"),
    temperature=0.2,
)

response = model.invoke("请给 LangChain 写一句适合初学者的学习建议。")

text = response.text.strip()

print("处理后的字符串：")
print(text)
```

运行：

```powershell
uv run examples/lesson_03_text_output.py
```

这里做了一个很小的处理：

```python
text = response.text.strip()
```

`strip()` 会去掉首尾空白字符。它不是 LangChain 特性，只是 Python 字符串方法。

很多业务场景下，模型输出最终都会进入这些位置：

- 命令行显示。
- Web 页面显示。
- 数据库存储。
- 日志记录。
- 下一个 LangChain 组件。

所以把输出变成稳定的字符串，是最基础的输出处理能力。

## 七、StrOutputParser：最简单的输出解析器

现在再看一个会在后续课程里反复出现的问题。

目前我们已经知道，可以这样拿到文本：

```python
response = model.invoke("请解释 LangChain。")
text = response.text
```

这在单个脚本里很清楚。

但当链路变长以后，代码会变成：

```text
Prompt 生成输入
  ↓
Model 返回 AIMessage
  ↓
手动取 response.text
  ↓
再传给下一个步骤
```

如果每一步都手动取 `.text`，会出现几个问题：

- “模型输出转字符串”这个动作散落在代码里。
- 后续组合 Prompt、Model、Parser 时，不够统一。
- 读代码时不容易看出哪一步负责解析输出。
- 将来从字符串解析升级到 JSON、Pydantic 结构时，不容易替换。

所以 LangChain 提供了 Output Parser，用来把模型输出转换成程序更容易使用的格式。

你可以把 Output Parser 理解成：

```text
模型输出对象 → 程序更想要的数据形态
```

有的 Parser 输出字符串，有的 Parser 输出 JSON，有的 Parser 输出结构化对象。

`StrOutputParser` 是最简单的一种，它的作用是：

> 从模型输出对象中提取文本，得到普通字符串。

也就是说，它把这个手动动作：

```python
text = response.text
```

变成一个可组合的 LangChain 组件：

```python
text = parser.invoke(response)
```

它来自：

```python
from langchain_core.output_parsers import StrOutputParser
```

这里使用 `langchain_core.output_parsers` 是合适的，因为当前官方参考文档仍然将输出解析器放在 `langchain_core` 中。

## 八、示例 3：单独使用 StrOutputParser

新建文件：

```text
examples/lesson_03_str_output_parser.py
```

代码：

```python
from os import getenv

from dotenv import load_dotenv
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI


load_dotenv()


model = ChatOpenAI(
    model=getenv("OPENAI_MODEL", "gpt-5-nano"),
    temperature=0.2,
)

parser = StrOutputParser()

message = model.invoke("请用一句话说明 PromptTemplate 的作用。")
text = parser.invoke(message)

print("原始消息对象类型：")
print(type(message))

print("\n解析后的文本类型：")
print(type(text))

print("\n解析后的文本：")
print(text)
```

运行：

```powershell
uv run examples/lesson_03_str_output_parser.py
```

观察重点：

- `message` 是模型返回的消息对象。
- `text` 是普通 Python 字符串。
- `StrOutputParser` 让“消息对象 → 字符串”这个动作变得明确。

## 九、示例 4：Prompt + Model + Parser

现在把第 2 课的 Prompt 模板和本课 Parser 连起来。

新建文件：

```text
examples/lesson_03_prompt_model_parser.py
```

代码：

```python
from os import getenv

from dotenv import load_dotenv
from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI


load_dotenv()


model = ChatOpenAI(
    model=getenv("OPENAI_MODEL", "gpt-5-nano"),
    temperature=0.2,
)

prompt = ChatPromptTemplate.from_messages(
    [
        ("system", "你是一名严谨的 LangChain 老师，回答要简洁清楚。"),
        ("human", "请用{style}风格解释这个概念：{concept}"),
    ]
)

parser = StrOutputParser()

prompt_value = prompt.invoke(
    {
        "style": "初学者能听懂的",
        "concept": "AIMessage",
    }
)

message = model.invoke(prompt_value)
text = parser.invoke(message)

print(text)
```

运行：

```powershell
uv run examples/lesson_03_prompt_model_parser.py
```

这段代码的数据流是：

```text
变量字典
  ↓
ChatPromptTemplate
  ↓
PromptValue
  ↓
ChatOpenAI
  ↓
AIMessage
  ↓
StrOutputParser
  ↓
str
```

这正是后面 Runnable 链式组合的基础。

下一阶段你会看到更简洁的写法：

```python
chain = prompt | model | parser
```

但这一课先不要急。现在先把每一步拆开看清楚。

## 十、基础文本后处理

有时我们不需要复杂解析，只需要一点普通字符串处理。

常见操作：

| 操作 | 示例 | 作用 |
| --- | --- | --- |
| 去除首尾空白 | `text.strip()` | 清理多余换行 |
| 按行拆分 | `text.splitlines()` | 得到每一行 |
| 判断是否包含关键词 | `"不确定" in text` | 做简单兜底 |
| 截断长度 | `text[:200]` | 控制展示长度 |
| 替换字符 | `text.replace("：", ":")` | 格式统一 |

### 示例 5：把模型输出拆成行

新建文件：

```text
examples/lesson_03_basic_text_processing.py
```

代码：

```python
from os import getenv

from dotenv import load_dotenv
from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI


load_dotenv()


model = ChatOpenAI(
    model=getenv("OPENAI_MODEL", "gpt-5-nano"),
    temperature=0.2,
)

prompt = ChatPromptTemplate.from_messages(
    [
        ("system", "你是一名技术学习教练。"),
        (
            "human",
            "请给学习 {topic} 的初学者列出 5 条建议。每条建议单独一行，不要编号。",
        ),
    ]
)

parser = StrOutputParser()

prompt_value = prompt.invoke({"topic": "LangChain"})
message = model.invoke(prompt_value)
text = parser.invoke(message)

suggestions = [
    line.strip()
    for line in text.splitlines()
    if line.strip()
]

print("建议列表：")
for index, suggestion in enumerate(suggestions, start=1):
    print(f"{index}. {suggestion}")
```

运行：

```powershell
uv run examples/lesson_03_basic_text_processing.py
```

观察重点：

- 这段代码依赖模型“每条建议单独一行”。
- 如果模型输出了编号、项目符号或额外解释，解析结果可能会不稳定。
- 这正是自由文本解析的局限。

## 十一、自由文本输出的局限

自由文本很适合给人读，但不一定适合给程序用。

例如你让模型输出：

```text
请提取姓名、邮箱、电话。
```

模型可能返回：

```text
姓名：张三
邮箱：zhangsan@example.com
电话：13800000000
```

也可能返回：

```text
以下是提取结果：
- 姓名: 张三
- 邮箱地址: zhangsan@example.com
- 联系电话: 13800000000
```

还可能返回：

```text
我找到的信息是张三，邮箱为 zhangsan@example.com，电话是 13800000000。
```

对人来说，这三种都能看懂。

对程序来说，它们是三种不同格式。

这会带来几个问题：

- 字段位置不固定。
- 标点和语言可能变化。
- 模型可能添加解释文字。
- 缺失字段不好判断。
- 后续系统很难稳定消费。

所以，在简单文本展示场景中，`StrOutputParser` 很好用；但在业务系统中，如果你需要稳定字段，就应该进入结构化输出。

这就是第 4 课要解决的问题。

## 十二、实践案例：文本摘要处理器

现在你来做一个实践案例：输入一段文本，模型生成摘要，然后程序做基础后处理。

新建文件：

```text
examples/lesson_03_summary_processor.py
```

代码：

```python
from os import getenv

from dotenv import load_dotenv
from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI


load_dotenv()


model = ChatOpenAI(
    model=getenv("OPENAI_MODEL", "gpt-5-nano"),
    temperature=0.2,
)

prompt = ChatPromptTemplate.from_messages(
    [
        (
            "system",
            "你是一名文本分析助手。请只根据用户提供的文本进行总结，不要补充外部事实。",
        ),
        (
            "human",
            """
请将下面文本总结成 3 条要点。
要求：
- 每条要点单独一行
- 不要输出标题
- 不要输出编号

文本：
{text}
""",
        ),
    ]
)

parser = StrOutputParser()

source_text = input("请输入要总结的文本：")

prompt_value = prompt.invoke({"text": source_text})
message = model.invoke(prompt_value)
summary_text = parser.invoke(message).strip()

summary_items = [
    line.strip("- ").strip()
    for line in summary_text.splitlines()
    if line.strip()
]

print("\n=== 摘要要点 ===\n")
for index, item in enumerate(summary_items, start=1):
    print(f"{index}. {item}")

print("\n=== 调试信息 ===\n")
print("原始模型输出：")
print(summary_text)
print("\nToken 用量：")
print(message.usage_metadata)
```

运行：

```powershell
uv run examples/lesson_03_summary_processor.py
```

建议输入：

```text
LangChain 是一个用于构建大语言模型应用的框架。它提供模型调用、Prompt 管理、工具调用、结构化输出、检索增强生成和 Agent 等能力。对于学习者来说，先理解消息、Prompt 和模型输出，再进入 RAG 和 Agent，会更容易建立清晰的工程思维。
```

观察重点：

- `summary_text` 是模型原始文本输出。
- `summary_items` 是程序从文本中拆出来的列表。
- 这种解析方式能用于简单场景，但仍然依赖模型遵守格式。
- `message.usage_metadata` 可以帮助你观察 token 用量。

## 十三、常见错误与排查

### 1. 把 `AIMessage` 当成字符串用

错误思路：

```python
response = model.invoke("你好")
result = response.strip()
```

`response` 是消息对象，不是字符串。

推荐写法：

```python
result = response.text.strip()
```

或：

```python
from langchain_core.output_parsers import StrOutputParser

parser = StrOutputParser()
result = parser.invoke(response).strip()
```

### 2. 过度依赖 `content` 的具体结构

简单文本场景中，`response.content` 常常是字符串。

但在多模态、工具调用、推理内容等场景中，`content` 可能包含更复杂的结构。

如果你只需要纯文本，优先使用：

```python
response.text
```

### 3. 元数据字段不稳定

不同模型供应商可能返回不同的 `response_metadata`。

不要假设所有模型都会有同一个字段，例如：

```python
response.response_metadata["finish_reason"]
```

更稳妥的写法是：

```python
finish_reason = response.response_metadata.get("finish_reason")
```

### 4. 用自由文本解析业务字段

如果你要提取姓名、邮箱、电话、金额、日期等字段，不建议长期依赖 `split()`、正则或固定行号。

第 4 课会学习更适合业务系统的结构化输出。

### 5. 忽略调试信息

当模型输出不符合预期时，不要只看最后展示给用户的内容。

至少检查：

- Prompt 最终是什么。
- 模型返回的 `AIMessage` 是什么。
- `text/content/content_blocks` 分别是什么。
- 是否有 `tool_calls`。
- token 用量是否异常。

## 十四、本课小结

这一课你完成了从“看模型回答”到“处理模型输出”的第一步：

- 聊天模型返回的通常是 `AIMessage`。
- `AIMessage` 不只是文本，还包含元数据、内容块、工具调用等信息。
- `response.text` 适合获取纯文本。
- `response.content` 是原始内容。
- `response.content_blocks` 是标准化内容块。
- `usage_metadata` 可以帮助观察 token 用量。
- `response_metadata` 可以帮助调试模型响应。
- `StrOutputParser` 可以把模型消息解析成普通字符串。
- 自由文本解析适合简单展示，不适合稳定业务字段。

请记住：

> LLM 应用开发不是只让模型说话，而是让程序能可靠地接住模型的输出。
