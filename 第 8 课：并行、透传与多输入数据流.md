# 第 8 课：并行、透传与多输入数据流

> 本课关键词：**RunnablePassthrough、RunnableParallel、透传、并行分支、多输入、多输出、中间变量**

## 一、本课目标

学完本课，你应该能做到：

1. 理解为什么线性链有时不够用。
2. 理解 `RunnablePassthrough` 的透传作用。

4. 使用 `RunnableParallel` 同时运行多个子链。
5. 理解多输入、多输出链路中的数据组织方式。
6. 构建一个同时生成摘要、标题和标签的文本分析链。

前面你已经能把 Prompt、模型和解析器组合成线性链。现在继续往真实项目靠近：当一条链需要保留原始输入、生成多个中间结果、再把这些结果组织成字典时，应该怎么写？

## 二、为什么线性链不够用

线性链很适合表达这样的流程：

```text
输入 → Prompt → Model → Parser → 输出
```

但真实应用经常不是一条直线。

例如你输入一段文章，希望同时得到：

- 标题。
- 摘要。
- 关键词标签。
- 原文长度。
- 后续审校建议。

如果只用线性链，你可能会写出很多分散变量：

```python
title = title_chain.invoke({"text": text})
summary = summary_chain.invoke({"text": text})
tags = tags_chain.invoke({"text": text})

result = {
    "text": text,
    "title": title,
    "summary": summary,
    "tags": tags,
}
```

这段代码不是不能用，但随着流程变复杂，会出现几个问题：

- 每个子链都要手动传同一份输入。
- 中间结果散落在变量里，链路结构不明显。
- 很难把整个流程封装成一个 Runnable。
- 后续如果要批量处理，代码会继续膨胀。
- 调试时，不容易看出每个分支来自哪里。

`RunnablePassthrough` 和 `RunnableParallel` 解决的就是这类数据流组织问题。

你可以先用一句话理解：

```text
RunnablePassthrough 用来保留或追加数据。
RunnableParallel 用来同时运行多个分支并收集结果。
```

---

## 三、什么是透传

透传的意思是：输入是什么，输出仍然是什么。

最小形式如下：

```text
input → RunnablePassthrough → input
```

这看起来好像“什么都没做”，但它在链路里很有用。

因为很多时候你不是想丢掉原始输入，而是想在原始输入旁边追加新字段：

```text
{"text": "原文"}
  ↓
追加 summary
  ↓
{"text": "原文", "summary": "摘要"}
```

如果没有透传能力，一条链每生成一个中间结果，就可能把前面的信息覆盖掉。

`RunnablePassthrough` 的设计意图，就是让你在链式组合中保留数据上下文。

---

## 四、示例 1：最小透传

先看最小例子。这个示例不调用模型，只观察 `RunnablePassthrough` 本身的行为。

新建文件：

```text
examples/lesson_08_passthrough_basic.py
```

代码：

```python
from langchain_core.runnables import RunnablePassthrough


passthrough = RunnablePassthrough()

result = passthrough.invoke(
    {
        "text": "LCEL 可以组织复杂数据流。",
        "source": "lesson_08",
    }
)

print(result)
print(result["text"])
```

运行：

```powershell
uv run examples/lesson_08_passthrough_basic.py
```

观察输出：

- 输入是一个字典。
- 输出仍然是同一个结构的字典。

这说明 `RunnablePassthrough()` 本身不会改变输入。它的价值主要体现在更复杂的链路组合里。

## 五、示例 2：使用 `assign()` 追加字段

`RunnablePassthrough.assign()` 可以在保留原始输入的基础上，给字典追加新字段。

先把它拆开理解：

```python
RunnablePassthrough.assign(
    新字段名=生成这个字段的逻辑
)
```

它通常用于字典输入。运行时，它会先保留原字典里的字段，再计算 `assign()` 中声明的新字段，最后返回一个合并后的新字典。

在下面的例子里：

- `text_length` 是追加的新字段。
- `preview` 也是追加的新字段。
- `lambda input_data: ...` 表示“根据当前输入字典计算这个字段的值”。

这里的 `lambda` 是 Python 的匿名函数写法。它适合写很短的临时函数：

```python
lambda input_data: len(input_data["text"])
```

等价于：

```python
def get_text_length(input_data):
    return len(input_data["text"])
```

在 `assign()` 里，LangChain 会把当前输入字典传给这个函数。比如输入是：

```python
{"text": "hello"}
```

那么：

```python
lambda input_data: len(input_data["text"])
```

就会返回 `5`。本例使用 `lambda`，只是为了让“如何计算新字段”直接写在 `assign()` 旁边。如果计算逻辑变复杂，应该改成有名字的普通函数。

所以它解决的问题是：你不需要手动创建一个新字典，也不需要把中间结果散落在临时变量里。

新建文件：

```text
examples/lesson_08_passthrough_assign.py
```

代码：

```python
from langchain_core.runnables import RunnablePassthrough


chain = RunnablePassthrough.assign(
    text_length=lambda input_data: len(input_data["text"]),
    preview=lambda input_data: input_data["text"][:12],
)

result = chain.invoke(
    {
        "text": "RunnablePassthrough 可以在链路中保留原始输入并追加新字段。"
    }
)

print(result)
```

运行：

```powershell
uv run examples/lesson_08_passthrough_assign.py
```

你会看到类似结构：

```python
{
    "text": "...",
    "text_length": 36,
    "preview": "RunnablePas..."
}
```

这里最重要的是数据形态变化：

```text
输入字典
  ↓
保留原字段
  ↓
追加 text_length 和 preview
```

这就是后面构建多步骤链时非常常见的模式。

如果用普通 Python 手写，大概会是：

```python
result = {
    **input_data,
    "text_length": len(input_data["text"]),
    "preview": input_data["text"][:12],
}
```

`assign()` 把这种“保留原输入 + 追加新字段”的模式放进 Runnable 链路里。

## 六、为什么需要保留原始输入

思考一个项目场景。

你先根据原文生成摘要，然后希望把“原文 + 摘要”一起交给下一步做质量检查。

如果摘要链只返回摘要，原文就丢了：

```text
{"text": "原文"} → 摘要链 → "摘要"
```

下一步想比较原文和摘要，就没有原文可用了。

更合理的数据流是：

```text
{"text": "原文"}
  ↓
追加 summary
  ↓
{"text": "原文", "summary": "摘要"}
  ↓
质量检查
```

`RunnablePassthrough.assign()` 让这个过程可以保留在 LCEL 链里，而不是散落成一堆临时变量。

## 七、示例 3：保留原文并追加摘要

这个示例会调用模型，并把生成的摘要追加回输入字典。

新建文件：

```text
examples/lesson_08_assign_summary.py
```

代码：

```python
from os import getenv

from dotenv import load_dotenv
from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import RunnablePassthrough
from langchain_openai import ChatOpenAI


load_dotenv()


model = ChatOpenAI(
    model=getenv("OPENAI_MODEL", "gpt-5-nano"),
    temperature=0.2,
)

parser = StrOutputParser()

summary_prompt = ChatPromptTemplate.from_messages(
    [
        ("system", "你是一名技术文档摘要助手。"),
        ("human", "请用一句话总结下面内容：\n\n{text}"),
    ]
)

summary_chain = summary_prompt | model | parser

chain = RunnablePassthrough.assign(summary=summary_chain)

result = chain.invoke(
    {
        "text": "RunnablePassthrough.assign 可以在保留原始输入的同时，把某个子链生成的结果追加为新字段。"
    }
)

print(result)
print("\n摘要：")
print(result["summary"])
```

运行：

```powershell
uv run examples/lesson_08_assign_summary.py
```

请观察：

- 输入里只有 `text`。
- 输出里仍然有 `text`。
- 输出里多了 `summary`。

这就是“保留上下文 + 追加中间结果”的典型写法。

## 八、什么是并行

并行在这里不是指你一定要关心底层线程或网络细节，而是指：

> 把同一个输入同时交给多个 Runnable 分支，并把每个分支的结果收集成一个字典。

可以这样理解：

```text
             → title_chain   → title
输入 text     → summary_chain → summary
             → tags_chain    → tags

最终输出：
{
    "title": ...,
    "summary": ...,
    "tags": ...
}
```

这正是 `RunnableParallel` 的用途。

它的设计意图是让多分支数据流仍然保持 Runnable 形态。组合后的并行链也可以继续用 `invoke()`、`batch()` 等统一接口。

## 九、示例 4：最小并行分支

先用普通 Python 函数包装成 Runnable，观察 `RunnableParallel` 的输出形态。

这里会用到 `RunnableLambda`。它可以把一个普通 Python 函数包装成 Runnable，让这个函数也能放进 LangChain 链路里。例子里的 `lambda input_data: ...` 仍然是刚才见过的匿名函数写法：接收输入字典，返回某个计算结果。

新建文件：

```text
examples/lesson_08_parallel_basic.py
```

代码：

```python
from langchain_core.runnables import RunnableLambda, RunnableParallel


parallel_chain = RunnableParallel(
    original=RunnableLambda(lambda input_data: input_data["text"]),
    length=RunnableLambda(lambda input_data: len(input_data["text"])),
    preview=RunnableLambda(lambda input_data: input_data["text"][:10]),
)

result = parallel_chain.invoke(
    {
        "text": "RunnableParallel 会把同一个输入交给多个分支。"
    }
)

print(result)
```

运行：

```powershell
uv run examples/lesson_08_parallel_basic.py
```

观察输出：

```python
{
    "original": "...",
    "length": 26,
    "preview": "Runnable..."
}
```

这里的关键是：

```text
每个分支接收同一个输入。
每个分支产生自己的输出。
RunnableParallel 把这些输出合并成字典。
```

## 十、示例 5：同时生成标题、摘要和标签

现在构建本课的核心实践案例。

目标：

```text
输入一段文本
  ↓
同时生成 title、summary、tags
  ↓
返回一个结果字典
```

这个示例会出现一个新写法：

```python
tags_prompt = tags_prompt.partial(
    format_instructions=tag_parser.get_format_instructions()
)
```

`partial()` 的意思是：先把 Prompt 模板中的一部分变量固定下来，得到一个新的 Prompt 模板。

在这里，`tags_prompt` 原本有两个变量：

```text
text
format_instructions
```

其中 `text` 每次运行都不同，应该由调用方传入；`format_instructions` 是关键词解析器给出的格式说明，通常是固定的，所以可以提前填进去。

这样做以后，调用链时只需要传：

```python
{"text": "..."}
```

而不必每次都手动传 `format_instructions`。这能减少重复参数，也能避免调用方忘记传格式说明。

新建文件：

```text
examples/lesson_08_parallel_text_analysis.py
```

代码：

```python
from os import getenv

from dotenv import load_dotenv
from langchain_core.output_parsers import CommaSeparatedListOutputParser, StrOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import RunnableParallel
from langchain_openai import ChatOpenAI


load_dotenv()


model = ChatOpenAI(
    model=getenv("OPENAI_MODEL", "gpt-5-nano"),
    temperature=0.2,
)

text_parser = StrOutputParser()
tag_parser = CommaSeparatedListOutputParser()

title_prompt = ChatPromptTemplate.from_messages(
    [
        ("system", "你是一名中文标题编辑。"),
        ("human", "请为下面文本生成一个不超过 18 个字的标题：\n\n{text}"),
    ]
)

summary_prompt = ChatPromptTemplate.from_messages(
    [
        ("system", "你是一名技术内容摘要助手。"),
        ("human", "请用一句话总结下面文本：\n\n{text}"),
    ]
)

tags_prompt = ChatPromptTemplate.from_messages(
    [
        ("system", "你只负责提取关键词，不要解释。"),
        (
            "human",
            "请从下面文本中提取 5 个关键词。\n\n"
            "文本：{text}\n\n"
            "{format_instructions}",
        ),
    ]
).partial(format_instructions=tag_parser.get_format_instructions())

title_chain = title_prompt | model | text_parser
summary_chain = summary_prompt | model | text_parser
tags_chain = tags_prompt | model | tag_parser

analysis_chain = RunnableParallel(
    title=title_chain,
    summary=summary_chain,
    tags=tags_chain,
)

result = analysis_chain.invoke(
    {
        "text": "LCEL 可以把多个 Runnable 组合成链。RunnableParallel 进一步允许同一个输入并行进入多个分支，并把结果组织成字典。"
    }
)

print(result)
```

运行：

```powershell
uv run examples/lesson_08_parallel_text_analysis.py
```

请观察：

- `title` 是字符串。
- `summary` 是字符串。
- `tags` 是列表。
- 三个分支共享同一个输入：`{"text": ...}`。
- `tags_prompt.partial(...)` 让标签链内部自己携带格式说明，调用方不需要关心这个细节。

这类结构很接近真实应用中的“文本分析流水线”。

## 十一、组合透传和并行

有时候你不仅要生成多个结果，还要保留原始输入。

可以先用 `RunnableParallel` 生成多个分析结果，再用 `RunnablePassthrough.assign()` 把它们追加到原始输入中。

下面的代码会把两个新写法合在一起：

- `partial(format_instructions=...)`：提前固定标签 Prompt 的格式说明。
- `RunnablePassthrough.assign(analysis=analysis_chain)`：保留原始 `text`，并追加一个 `analysis` 字段。

新建文件：

```text
examples/lesson_08_passthrough_with_parallel.py
```

代码：

```python
from os import getenv

from dotenv import load_dotenv
from langchain_core.output_parsers import CommaSeparatedListOutputParser, StrOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import RunnableParallel, RunnablePassthrough
from langchain_openai import ChatOpenAI


load_dotenv()


model = ChatOpenAI(
    model=getenv("OPENAI_MODEL", "gpt-5-nano"),
    temperature=0.2,
)

text_parser = StrOutputParser()
tag_parser = CommaSeparatedListOutputParser()

summary_prompt = ChatPromptTemplate.from_template(
    "请用一句话总结下面文本：\n\n{text}"
)

title_prompt = ChatPromptTemplate.from_template(
    "请为下面文本生成一个不超过 18 个字的标题：\n\n{text}"
)

tags_prompt = ChatPromptTemplate.from_template(
    "请从下面文本中提取 5 个关键词。\n\n"
    "文本：{text}\n\n"
    "{format_instructions}"
).partial(format_instructions=tag_parser.get_format_instructions())

analysis_chain = RunnableParallel(
    title=title_prompt | model | text_parser,
    summary=summary_prompt | model | text_parser,
    tags=tags_prompt | model | tag_parser,
)

chain = RunnablePassthrough.assign(analysis=analysis_chain)

result = chain.invoke(
    {
        "text": "在复杂链路中，透传可以保留原始输入，并行分支可以同时生成多个中间结果。"
    }
)

print(result["text"])
print(result["analysis"])
```

运行：

```powershell
uv run examples/lesson_08_passthrough_with_parallel.py
```

输出结构大致是：

```python
{
    "text": "原文",
    "analysis": {
        "title": "...",
        "summary": "...",
        "tags": [...]
    }
}
```

这个结构比把所有字段平铺在一起更清晰，因为它保留了层次：

```text
原始输入：text
分析结果：analysis
```

## 十二、多输入、多输出的数据组织思维

到了这一课，真正重要的已经不是类名，而是数据形态。

你应该开始养成一个习惯：每写一条链，都先画出输入输出。

例如：

```text
输入：
{
    "text": str
}

输出：
{
    "text": str,
    "analysis": {
        "title": str,
        "summary": str,
        "tags": list[str]
    }
}
```

这种写法看起来像文档，但它会直接影响代码质量。

因为复杂链路真正容易出错的地方往往不是模型调用，而是：

- 某个字段名写错。
- 某个分支需要 `text`，但输入里没有。
- 上一步输出的是字符串，下一步却期待字典。
- 中间结果被覆盖，后面步骤拿不到原始输入。
- 所有结果都平铺在顶层，后续维护时不知道字段来自哪里。

`RunnablePassthrough` 和 `RunnableParallel` 让你可以用 LCEL 明确表达这些数据结构。

---

## 十三、什么时候使用哪一个

可以用下面的规则快速判断：

| 需求 | 推荐方式 |
| --- | --- |
| 原样保留输入 | `RunnablePassthrough()` |
| 在输入字典上追加字段 | `RunnablePassthrough.assign(...)` |
| 同一个输入进入多个分支 | `RunnableParallel(...)` |
| 多个分支结果需要合并成字典 | `RunnableParallel(...)` |
| 先保留原文，再追加分析结果 | `RunnablePassthrough.assign(analysis=parallel_chain)` |

如果你只是简单的：

```text
Prompt → Model → Parser
```

那第 6 课的线性 LCEL 就够了。

如果你开始需要保留、追加、并行、组合多个结果，就进入了本课的范围。

## 十四、常见错误与排查

### 1. 以为 `RunnablePassthrough` 会自动生成字段

`RunnablePassthrough()` 只会原样传递输入。

如果要追加字段，需要使用：

```python
RunnablePassthrough.assign(new_key=some_chain)
```

### 2. 给 `assign()` 传入非字典输入

`assign()` 通常用于字典输入，因为它要在原字典上追加 key。

如果你传入的是普通字符串，就不适合直接追加字段。

更稳妥的方式是先把输入组织成字典：

```python
chain.invoke({"text": "..."})
```

### 3. 并行分支需要不同输入，但你没有整理输入结构

`RunnableParallel` 会把同一个输入交给每个分支。

如果某个分支需要 `text`，另一个分支需要 `question`，你要确保输入字典同时包含这些字段：

```python
{
    "text": "...",
    "question": "..."
}
```

### 4. 字段名冲突

如果原始输入里已经有 `summary`，你又写：

```python
RunnablePassthrough.assign(summary=summary_chain)
```

新的 `summary` 可能覆盖旧字段。

更清晰的写法是把分析结果放到嵌套字段里：

```python
RunnablePassthrough.assign(analysis=analysis_chain)
```

### 5. 把并行理解成一定更快

`RunnableParallel` 的重点是表达多个分支的数据流。

它通常有助于组织并行执行，但真实速度还会受到模型服务、网络、限流、分支数量等因素影响。写教材和项目代码时，不要把它简单理解成“用了就一定更快”。

### 6. 输出结构太复杂却没有打印检查

复杂链路写完后，先打印结果结构：

```python
print(result.keys())
print(result)
```

如果结果嵌套较深，可以先用小输入测试，再接入真实业务数据。

## 十五、本课小结

这一课你从线性链进入了多分支数据流。

你需要记住：

- `RunnablePassthrough` 用于透传输入。
- `RunnablePassthrough.assign()` 可以在保留原输入的基础上追加字段。
- `RunnableParallel` 可以把同一个输入交给多个分支。
- `RunnableParallel` 的输出是一个字典，key 是分支名，value 是分支结果。
- 透传适合保留上下文，并行适合生成多个中间结果。
- 多输入、多输出链路最重要的是先设计清楚数据结构。

第 6 课解决“如何串起来”，第 7 课解决“如何解析结果”，本课解决“如何组织多个结果和中间变量”。

### 练习 1：透传输入

创建：

```text
examples/lesson_08_passthrough_practice.py
```

- 使用 `RunnablePassthrough()` 原样返回。

### 练习 2：追加统计字段

```text

```

要求：

- 使用 `RunnablePassthrough.assign()`。
- 给输入字典追加 `text_length` 和 `has_langchain` 两个字段。

### 练习 4：保留原文和分析结果

