# LangChain 1.x 系统学习路线总纲

> 主线：**基础模型调用 → Runnable 与结构化输出 → RAG → Tools → Agent → LangGraph → 调试评估 → 工程化项目 → Deep Agents 拓展**

本路线面向希望系统掌握 LangChain 1.x 生态的学习者。课程采用“概念讲解 + 小案例 + 阶段项目 + 工程复盘”的方式，从最小可运行程序逐步推进到 RAG、Agent、LangGraph 工作流和复杂智能体系统设计。

---

## 一、课程定位

### 适合人群

- 已具备 Python 基础，能读写函数、类、类型标注和常见项目结构。
- 想系统学习 LangChain 1.x，而不是只复制零散示例。
- 想从 Prompt 调用进阶到 RAG、工具调用、Agent、LangGraph 和项目工程化。
- 希望最终能独立开发一个可演示、可维护、可迭代的 LLM 应用。

### 前置要求

- Python 3.11+ 基础。
- 基本命令行能力。
- 了解 HTTP API、JSON、环境变量和虚拟环境。
- 有一个可调用的聊天模型服务，例如 OpenAI 兼容接口、本地模型网关或其他模型供应商。

### 课程原则

- **先稳定，再复杂**：先掌握模型调用、结构化输出和可控链路，再进入 Agent。
- **先数据流，再智能体**：先理解输入输出、状态和错误边界，再让模型自主调用工具。
- **先可观察，再优化**：先能看到调用过程、检索结果和中间状态，再谈调优。
- **每阶段有作品**：每个阶段都产出一个小项目或项目模块，避免只学 API 名称。
- **以 LangChain 1.x 为主线**：重点关注 `langchain`、`langchain-core`、`create_agent`、结构化输出、工具调用、LangGraph 和 LangSmith；旧版链路只在必要时解释迁移背景。

### 教材准确性约定

- 每一课都以 LangChain 1.x 最新官方文档和当前 SDK 推荐写法为基准。
- 示例代码优先使用 1.x 推荐的公开入口，避免沿用旧版教程中的过时代码。
- 运行环境统一使用 `uv`，代码运行命令统一使用 `uv run ...`。
- 模型名称统一通过 `.env` 配置，例如 `OPENAI_MODEL=gpt-5-nano`，避免把教材绑定到某个固定模型版本。
- 当官方 API 或推荐实践变化时，应优先修订课程讲义和示例代码。

# 阶段一：LangChain 1.x 入门与基础调用

**课时数：4 课**

## 阶段目标

建立 LangChain 1.x 的整体认知，完成开发环境搭建，掌握聊天模型、消息、Prompt 模板和基础结构化输出。

## 核心知识点

- LangChain、LangGraph、LangSmith、Deep Agents 的定位。
- LangChain 1.x 与旧版 LangChain 的核心差异。
- Chat Model 调用。
- System / Human / AI Messages。
- `PromptTemplate` 与 `ChatPromptTemplate`。
- `invoke()`、`stream()` 的基础使用。
- 结构化输出的基本价值。

## 阶段项目

**学习概念解释助手**

输入一个技术概念，输出：

- 通俗解释。
- 核心要点。
- 一个真实类比。
- 一个练习题。

## 验收标准

- 能独立搭建虚拟环境并安装依赖。
- 能通过模型完成一次稳定的聊天调用。
- 能使用 Prompt 模板组织变量。
- 能解释 Messages、Prompt、Model、Output 之间的关系。

## 课程安排

### 第 1 课：LangChain 1.x 全景认知与环境搭建

**该课目标：**

- 理解 LangChain 生态中各组件的职责边界。
- 完成 Python、虚拟环境、依赖和 API Key 配置。
- 跑通第一个模型调用程序。
- 理解 `invoke()` 的输入输出。

**案例：** 第一个命令行问答助手。

### 第 2 课：Messages 与 Prompt 模板

**该课目标：**

- 理解 System、Human、AI Message 的作用。
- 掌握 `PromptTemplate` 和 `ChatPromptTemplate`。
- 使用变量动态生成提示词。
- 初步形成“提示词即接口”的意识。

**案例：** 技术概念解释器。

### 第 3 课：模型输出与基础解析

**该课目标：**

- 理解模型返回对象与文本内容的区别。
- 掌握字符串输出解析的基本方式。
- 认识自由文本输出的局限。
- 为结构化输出做铺垫。

**案例：** 文本摘要与关键词提取工具。

### 第 4 课：结构化输出初步实践

**该课目标：**

- 理解为什么业务系统需要结构化输出。
- 使用 Pydantic 定义输出结构。
- 掌握基础 `with_structured_output` 思路。
- 实现一个信息提取器雏形。

**案例：** 招聘信息提取器。

# 阶段二：Runnable、LCEL 与链式编排

**课时数：5 课**

## 阶段目标

掌握 LangChain 1.x 的核心组合方式，建立“组件有输入输出、链路可组合、结果可追踪”的开发思维。

## 核心知识点

- Runnable 统一接口。
- LCEL 链式组合：`prompt | model | parser`。
- `invoke / batch / stream`。
- `RunnableLambda`、`RunnablePassthrough`、`RunnableParallel`。
- 输出解析器。
- 多步骤链路与中间变量传递。
- 可复用链路封装。

## 阶段项目

**多功能文本处理台**

支持：

- 摘要。
- 改写。
- 关键词提取。
- 中英翻译。
- 批量处理。
- 流式输出。

## 验收标准

- 能解释 Runnable 为什么是 LangChain 的基础抽象。
- 能组合 Prompt、Model、Parser。
- 能使用批处理与流式输出。
- 能将多步骤处理封装成可复用函数或模块。

## 课程安排

### 第 5 课：Runnable 核心接口深入

**该课目标：**

- 理解 Runnable 的输入输出约定。
- 掌握 `invoke()`、`batch()`、`stream()` 的适用场景。
- 理解“所有组件统一组合”的价值。

**案例：** 单步文本改写链。

### 第 6 课：LCEL 链式组合

**该课目标：**

- 掌握 `|` 管道组合方式。
- 理解链路中每一段的职责。
- 构建 `prompt | model | parser` 基础链。

**案例：** 摘要生成链。

### 第 7 课：输出解析与格式转换

**该课目标：**

- 理解 Output Parser 的作用。
- 将模型输出转换为字符串、列表、JSON 或业务对象。
- 明确 Parser 在链路中的位置。

**案例：** 关键词列表提取器。

### 第 8 课：并行、透传与多输入数据流

**该课目标：**

- 掌握 `RunnablePassthrough` 的用途。
- 掌握并行生成多个中间结果。
- 理解多输入、多输出链路的数据组织方式。

**案例：** 同时生成摘要、标题和标签。

### 第 9 课：可复用链路封装

**该课目标：**

- 把 Prompt、Model、Parser 封装为业务函数。
- 设计清晰的输入输出结构。
- 为后续项目建立基础目录结构。

**案例：** 文本处理台项目整理。

# 阶段三：结构化输出与程序化决策

**课时数：4 课**

## 阶段目标

从“模型能回答”升级到“模型能稳定产出程序可消费的数据”，掌握抽取、分类、路由和校验能力。

## 核心知识点

- Pydantic schema 设计。
- 字段描述、枚举、可选字段、列表字段。
- 信息抽取。
- 分类、打标、评分。
- 路由决策。
- 结果校验、兜底与错误恢复。

## 阶段项目

**工单智能分发器**

输入用户工单，输出：

- 工单类型。
- 紧急程度。
- 责任部门。
- 需要补充的信息。
- 推荐处理动作。

## 验收标准

- 能设计稳定的结构化输出 schema。
- 能将自然语言转为业务字段。
- 能用结构化结果驱动程序分支。
- 能处理字段缺失、分类不确定和模型输出异常。

## 课程安排

### 第 10 课：Pydantic 结构建模与字段设计

**该课目标：**

- 设计适合 LLM 输出的 schema。
- 正确使用字段描述、枚举和可选字段。
- 避免过度复杂的输出结构。

**案例：** 简历信息 schema 设计。

### 第 11 课：信息抽取器实战

**该课目标：**

- 从自然语言中抽取固定字段。
- 处理列表、日期、联系方式等常见字段。
- 构建可复用抽取链。

**案例：** 简历信息提取器。

### 第 12 课：分类、打标与路由

**该课目标：**

- 构建文本分类器。
- 输出可用于程序分支判断的结构化结果。
- 将分类结果连接到不同处理链路。

**案例：** 工单分类与部门路由。

### 第 13 课：结构化结果校验与兜底

**该课目标：**

- 理解结构化输出不是绝对可靠。
- 增加字段校验、默认值和异常处理。
- 设计失败重试与人工复核策略。

**案例：** 不完整工单的补充信息识别。

# 阶段四：RAG 知识库应用开发

**课时数：7 课**

## 阶段目标

系统掌握 RAG 的完整流程，从文档加载、清洗、切分、向量化、检索到问答生成和质量优化。

## 核心知识点

- RAG 基本原理。
- 文档加载与元数据。
- 文本清洗与切分。
- Embeddings。
- 向量数据库。
- Retriever。
- 检索增强生成链。
- 引用来源与上下文透明。
- 召回、重排、查询改写与 Prompt 优化。

## 阶段项目

**本地知识库问答助手**

支持：

- 导入 Markdown、TXT、PDF。
- 多文档检索。
- 返回答案与引用片段。
- 显示检索到的上下文。
- 支持基础质量调优。

## 验收标准

- 能完整讲清 RAG 的数据流。
- 能独立搭建小型知识库。
- 能定位答案错误是来自检索、上下文还是生成。
- 能通过切分、top-k、Prompt 和元数据过滤做基础优化。

## 课程安排

### 第 14 课：RAG 全景理解与适用边界

**该课目标：**

- 理解 RAG 解决什么问题。
- 区分模型知识、外部知识和业务知识。
- 明确 RAG 不适合解决的问题。

**案例：** 对比无 RAG 与有 RAG 的回答差异。

### 第 15 课：文档加载、清洗与元数据

**该课目标：**

- 加载 TXT、Markdown、PDF 文档。
- 提取来源、标题、页码等元数据。
- 处理乱码、空行、页眉页脚等噪声。

**案例：** 企业制度文档加载器。

### 第 16 课：文本切分策略

**该课目标：**

- 理解 chunk size 与 overlap。
- 掌握按字符、标题、语义边界切分的差异。
- 避免切分过碎或过长。

**案例：** 对比不同切分策略的检索效果。

### 第 17 课：Embeddings 与向量数据库

**该课目标：**

- 理解向量化和相似度检索。
- 将文档写入向量库。
- 完成基础相似度查询。

**案例：** 构建第一个文档索引。

### 第 18 课：Retriever 与检索结果分析

**该课目标：**

- 掌握 Retriever 的作用。
- 查看每次查询召回了哪些片段。
- 理解“检索结果不是最终答案”。

**案例：** 知识库上下文查看器。

### 第 19 课：构建 RAG 问答链

**该课目标：**

- 串联查询、检索、Prompt、模型和解析器。
- 要求答案基于上下文。
- 返回引用来源。

**案例：** 企业文档问答助手。

### 第 20 课：RAG 优化与阶段项目

**该课目标：**

- 识别召回不足、上下文污染、幻觉和答案不完整。
- 调整 chunk、top-k、Prompt、元数据过滤。
- 完成多文档知识库项目。

**案例：** 本地知识库问答助手完整实现。

# 阶段六：LangGraph 状态工作流

**课时数：5 课**

掌握 LangGraph 的核心思想，学会构建可控、可恢复、可观测的多步骤 Agent 或工作流。

## 核心知识点

- LangGraph 与 LangChain Agent 的关系。
- StateGraph。
- 状态对象。
- 节点与边。
- 条件分支。
- 循环与重试。
- Checkpoint 与持久化。
- Human-in-the-loop。
- 长任务与可恢复执行。

## 阶段项目

**工单处理工作流**

流程包括：

- 接收工单。
- 分类与补充信息判断。
- 检索知识库。
- 生成处理建议。
- 人工确认。
- 输出最终处理记录。

## 验收标准

- 能设计状态对象和节点职责。
- 能使用条件边控制流程。
- 能实现循环、重试或人工确认。
- 能解释 LangGraph 适合解决什么复杂度的问题。

## 课程安排

### 第 26 课：LangGraph 是什么，为什么需要状态编排

**该课目标：**

- 理解 LangGraph 的定位。
- 区分简单 Agent 与自定义工作流。
- 建立状态驱动的编排思维。

**案例：** 固定三步分析流程。

### 第 27 课：StateGraph、节点与状态对象

**该课目标：**

- 定义状态结构。
- 编写节点函数。
- 理解节点如何读取和写入状态。

**案例：** 文档分析图。

### 第 28 课：条件分支与路由控制

**该课目标：**

- 根据结构化结果选择下一步。
- 实现不同业务路径。
- 避免把所有判断都交给自由文本。

### 第 29 课：循环、重试与人工介入

**该课目标：**

## 六、学习建议

- 不要急着进入 Agent。先把 Runnable、结构化输出和 RAG 的数据流练熟。
- 每个项目都要打印或记录中间结果，包括 Prompt、检索片段、工具参数和结构化输出。
- RAG 阶段重点练“问题定位”：答案错了，先判断是没检索到、检索到了但没用好，还是模型生成时偏离上下文。
- Agent 阶段重点练“边界控制”：工具越强，越需要权限、确认、日志和失败处理。
- LangGraph 阶段重点练“状态设计”：好的状态结构比堆更多节点更重要。
- 工程化阶段重点练“可复现”：换一台机器、换一个同学，也能按 README 跑起来。
