# 第 11 课：信息抽取器实战

> 本课关键词：**信息抽取、简历抽取器、固定字段、列表字段、日期字段、联系方式、可复用抽取链**

## 一、本课目标

学完本课，你应该能做到：

1. 理解信息抽取器和普通问答链的区别。
2. 从自然语言文本中抽取固定字段。
3. 处理姓名、联系方式、日期、列表等常见字段。
4. 设计一个简历信息抽取 schema。
5. 使用 `with_structured_output()` 构建可复用抽取链。
6. 把抽取链封装成业务函数，方便后续项目复用。

第 10 课重点练了 schema 设计。这一课把 schema 用在一个完整任务上：给一段简历文本，输出程序可以直接使用的结构化简历信息。

## 二、什么是信息抽取

信息抽取的目标不是让模型“解释一段文本”，而是从文本中找出指定信息，并放进固定字段里。

例如输入：

```text
李明，5 年 Python 后端开发经验，熟悉 FastAPI、PostgreSQL、Redis。
邮箱 liming@example.com，电话 13800001111。
```

你希望输出：

```python
{
    "candidate_name": "李明",
    "years_of_experience": 5,
    "email": "liming@example.com",
    "phone": "13800001111",
    "core_skills": ["Python", "FastAPI", "PostgreSQL", "Redis"]
}
```

这和普通问答不同。

普通问答更关注“回答得是否自然”；信息抽取更关注：

- 字段是否完整。
- 字段类型是否正确。
- 缺失信息是否用 `None` 表达。
- 列表是否真的返回列表。
- 日期和联系方式是否保持可处理格式。
- 输出是否能进入数据库、路由、筛选或报表。

信息抽取器的价值，就是把自然语言文本变成业务系统能继续处理的数据。

---

## 三、抽取器的基本形态

一个结构化信息抽取器通常由三部分组成：

```text
Schema：定义要抽取什么字段
Prompt：说明抽取规则和边界
Model：根据 schema 返回结构化对象
```

在代码中会变成：

```python
structured_model = model.with_structured_output(ResumeExtraction)
result = structured_model.invoke(prompt_text)
```

这里的重点不是再记一次调用方式，而是理解抽取器的输入输出契约：

```text
输入：一段自然语言文本
输出：一个字段稳定的 Pydantic 对象
```

只要这个契约稳定，后续你就可以把抽取结果交给筛选器、分类器、数据库写入逻辑或人工审核页面。

## 四、简历抽取字段怎么设计

简历抽取常见字段包括：

| 字段类型 | 示例字段 | 设计建议 |
| --- | --- | --- |
| 基础信息 | 姓名、邮箱、电话 | 可能缺失，建议可选 |
| 日期信息 | 毕业日期、最近工作开始日期 | 建议统一格式，例如 `YYYY-MM` 或 `YYYY-MM-DD` |
| 数值信息 | 工作年限 | 用 `int | None`，无法判断时为 `None` |
| 列表信息 | 技能、项目、证书 | 用 `list[str]` |
| 判断信息 | 是否匹配某岗位 | 后续分类课会深入，本课只做简单匹配字段 |

日期字段要特别小心。

模型看到“2021 年 6 月”“2021.06”“2021/06”时，可能按不同格式输出。为了后续程序好处理，字段描述里要明确格式：

```python
graduation_date: str | None = Field(
    default=None,
    description="毕业日期，统一使用 YYYY-MM 格式；无法判断时为 None",
)
```

联系方式也类似。不要让模型生成解释性文字，只抽取原文中出现的邮箱和电话：

```python
email: str | None = Field(
    default=None,
    description="候选人邮箱，只提取原文中明确出现的邮箱；没有则为 None",
)
```

## 五、示例 1：最小简历抽取器

先做一个最小可运行版本。

新建文件：

```text
examples/lesson_11_minimal_resume_extractor.py
```

代码：

```python
from os import getenv

from dotenv import load_dotenv
from langchain_openai import ChatOpenAI
from pydantic import BaseModel, Field


load_dotenv()


class MinimalResume(BaseModel):
    candidate_name: str | None = Field(
        default=None,
        description="候选人姓名，如果原文中没有出现则为 None",
    )
    email: str | None = Field(
        default=None,
        description="候选人邮箱，只提取原文中明确出现的邮箱；没有则为 None",
    )
    phone: str | None = Field(
        default=None,
        description="候选人电话，只提取原文中明确出现的电话号码；没有则为 None",
    )
    core_skills: list[str] = Field(
        description="候选人的核心技能列表，只提取原文中明确出现的技能"
    )


model = ChatOpenAI(
    model=getenv("OPENAI_MODEL", "gpt-5-nano"),
    temperature=0,
)

extractor = model.with_structured_output(MinimalResume)

resume_text = """
李明，后端开发工程师。熟悉 Python、FastAPI、PostgreSQL、Redis。
邮箱：liming@example.com，电话：13800001111。
"""

result = extractor.invoke(
    f"请从下面简历文本中抽取候选人基础信息：\n\n{resume_text}"
)

print(result)
print(result.candidate_name)
print(result.core_skills)
```

运行：

```powershell
uv run examples/lesson_11_minimal_resume_extractor.py
```

请观察：

- `candidate_name`、`email`、`phone` 可能是字符串，也可能是 `None`。
- `core_skills` 是 `list[str]`，不是一段逗号分隔文本。
- `temperature=0` 用来降低格式和判断的随机性。

这个示例已经是一个最小信息抽取器。

## 六、为什么联系方式字段通常用字符串

你可能会问：电话为什么不用 `int`？

因为电话号码不是用于数学计算的数字。它可能包含：

- 前导 0。
- 国家区号。
- 分机号。
- 空格或短横线。

例如：

```text
010-88886666
+86 13800001111
021-12345678-8001
```

如果用 `int`，这些格式信息可能丢失。

所以联系方式建议先用字符串：

```python
phone: str | None
email: str | None
```

更严格的格式校验可以放到后续校验与兜底环节。本课先把重点放在稳定抽取字段。

## 七、示例 2：处理日期、年限和项目列表

现在扩展 schema，加入日期、工作年限和项目列表。

新建文件：

```text
examples/lesson_11_resume_extractor_with_dates.py
```

代码：

```python
from os import getenv

from dotenv import load_dotenv
from langchain_openai import ChatOpenAI
from pydantic import BaseModel, Field


load_dotenv()


class ProjectExperience(BaseModel):
    project_name: str = Field(description="项目名称")
    role: str | None = Field(
        default=None,
        description="候选人在项目中的角色或职责；无法判断时为 None",
    )
    technologies: list[str] = Field(
        description="项目中明确提到的技术栈列表"
    )


class ResumeExtraction(BaseModel):
    candidate_name: str | None = Field(
        default=None,
        description="候选人姓名，如果原文中没有出现则为 None",
    )
    graduation_date: str | None = Field(
        default=None,
        description="毕业日期，统一使用 YYYY-MM 格式；无法判断时为 None",
    )
    years_of_experience: int | None = Field(
        default=None,
        description="候选人总工作年限，只填写整数年；无法判断时为 None",
    )
    core_skills: list[str] = Field(
        description="候选人的核心技能列表，只提取原文中明确出现的技能"
    )
    projects: list[ProjectExperience] = Field(
        description="候选人的项目经历列表，只提取原文中明确出现的项目"
    )


model = ChatOpenAI(
    model=getenv("OPENAI_MODEL", "gpt-5-nano"),
    temperature=0,
)

extractor = model.with_structured_output(ResumeExtraction)

resume_text = """
候选人：王芳
2019 年 6 月毕业于某大学软件工程专业。
拥有 4 年后端开发经验，熟悉 Python、Django、MySQL、Docker。
项目经历：
1. 客户管理系统：负责后端接口开发，使用 Django 和 MySQL。
2. 报表自动化平台：负责数据处理模块，使用 Python 和 Docker。
"""

result = extractor.invoke(
    "请从下面简历文本中抽取结构化信息。日期字段请统一为 YYYY-MM 格式。\n\n"
    f"{resume_text}"
)

print(result)
print(result.graduation_date)
print(result.projects)
```

运行：

```powershell
uv run examples/lesson_11_resume_extractor_with_dates.py
```

这个示例出现了一个新结构：嵌套 Pydantic 模型。

```python
class ProjectExperience(BaseModel):
    ...

projects: list[ProjectExperience]
```

它表示 `projects` 不是普通字符串列表，而是“项目对象列表”。每个项目对象都有自己的字段：`project_name`、`role`、`technologies`。

嵌套模型适合这种场景：

```text
一个字段里有多条记录；
每条记录又有多个子字段。
```

例如项目经历、教育经历、工作经历都可能用到这种结构。

但嵌套不要过深。两层以内通常还比较容易控制；如果你一开始就设计很多层嵌套，模型更容易漏填或混淆字段。

## 八、抽取规则要写清楚

抽取任务里，Prompt 的重点不是“让模型发挥”，而是明确边界。

比较弱的写法：

```text
请提取简历信息。
```

更好的写法：

```text
请只根据原文中明确出现的信息抽取字段。
不要编造没有出现的信息。
日期统一使用 YYYY-MM 格式。
无法判断的可选字段返回 None。
技能列表只包含原文中明确出现的技能词。
```

这里的规则和 schema 是配合关系：

- schema 定义字段形态。
- Prompt 定义抽取边界。
- 模型负责把原文映射到字段。

如果字段描述已经写得很清楚，Prompt 可以更短；如果任务边界比较复杂，Prompt 就需要补充规则。

## 九、示例 3：封装可复用抽取链

现在把抽取器封装成函数，方便后续项目调用。

新建文件：

```text
examples/lesson_11_reusable_resume_extractor.py
```

代码：

```python
from os import getenv

from dotenv import load_dotenv
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from pydantic import BaseModel, Field


class ProjectExperience(BaseModel):
    project_name: str = Field(description="项目名称")
    role: str | None = Field(
        default=None,
        description="候选人在项目中的角色或职责；无法判断时为 None",
    )
    technologies: list[str] = Field(
        description="项目中明确提到的技术栈列表"
    )


class ResumeExtraction(BaseModel):
    candidate_name: str | None = Field(
        default=None,
        description="候选人姓名，如果原文中没有出现则为 None",
    )
    email: str | None = Field(
        default=None,
        description="候选人邮箱，只提取原文中明确出现的邮箱；没有则为 None",
    )
    phone: str | None = Field(
        default=None,
        description="候选人电话，只提取原文中明确出现的电话号码；没有则为 None",
    )
    graduation_date: str | None = Field(
        default=None,
        description="毕业日期，统一使用 YYYY-MM 格式；无法判断时为 None",
    )
    years_of_experience: int | None = Field(
        default=None,
        description="候选人总工作年限，只填写整数年；无法判断时为 None",
    )
    core_skills: list[str] = Field(
        description="候选人的核心技能列表，只提取原文中明确出现的技能"
    )
    projects: list[ProjectExperience] = Field(
        description="候选人的项目经历列表，只提取原文中明确出现的项目"
    )


def create_model() -> ChatOpenAI:
    return ChatOpenAI(
        model=getenv("OPENAI_MODEL", "gpt-5-nano"),
        temperature=0,
    )


def build_resume_extractor(model: ChatOpenAI):
    prompt = ChatPromptTemplate.from_messages(
        [
            (
                "system",
                "你是一个严谨的信息抽取器，只根据原文中明确出现的信息填写字段。",
            ),
            (
                "human",
                "请从下面简历文本中抽取结构化信息。\n"
                "规则：\n"
                "1. 不要编造原文没有出现的信息。\n"
                "2. 日期统一使用 YYYY-MM 格式。\n"
                "3. 无法判断的可选字段返回 None。\n"
                "4. 技能和项目只提取原文中明确出现的内容。\n\n"
                "简历文本：\n{resume_text}",
            ),
        ]
    )

    structured_model = model.with_structured_output(ResumeExtraction)
    return prompt | structured_model


def extract_resume(resume_text: str) -> ResumeExtraction:
    model = create_model()
    extractor = build_resume_extractor(model)
    return extractor.invoke({"resume_text": resume_text})


def main() -> None:
    load_dotenv()

    resume_text = """
    赵强，邮箱 zhaoqiang@example.com，电话 13900002222。
    2020 年 7 月本科毕业，3 年 Python 后端开发经验。
    熟悉 FastAPI、PostgreSQL、Celery、Docker。
    项目：任务调度平台，负责后端接口和异步任务模块开发，使用 FastAPI、Celery。
    """

    result = extract_resume(resume_text)

    print(result)
    print(result.email)
    print(result.projects[0].technologies)


if __name__ == "__main__":
    main()
```

运行：

```powershell
uv run examples/lesson_11_reusable_resume_extractor.py
```

这里的链路是：

```text
{"resume_text": "..."}
  ↓
ChatPromptTemplate
  ↓
structured_model
  ↓
ResumeExtraction
```

`prompt | structured_model` 是一个很实用的抽取器形态：Prompt 负责抽取规则，结构化模型负责按 schema 返回对象。

## 十、为什么抽取链要封装

抽取器通常不会只用一次。

后续你可能会把它接到：

- 命令行工具。
- Web API。
- 批量简历处理脚本。
- 候选人筛选链。
- 数据库写入逻辑。

如果抽取逻辑散在脚本里，就很难复用。

封装后，你只需要调用：

```python
result = extract_resume(resume_text)
```

或者复用链对象：

```python
extractor = build_resume_extractor(model)
results = extractor.batch(inputs)
```

这也是前面学习 Runnable 和链路封装的价值：抽取器不是一段一次性脚本，而是一个可以接入项目的数据处理能力。

## 十一、示例 4：批量抽取简历

如果你有多段简历文本，可以用同一条抽取链批量处理。

新建文件：

```text
examples/lesson_11_batch_resume_extraction.py
```

代码：

```python
from os import getenv

from dotenv import load_dotenv
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from pydantic import BaseModel, Field


class ResumeSummary(BaseModel):
    candidate_name: str | None = Field(
        default=None,
        description="候选人姓名，如果原文中没有出现则为 None",
    )
    years_of_experience: int | None = Field(
        default=None,
        description="候选人总工作年限，只填写整数年；无法判断时为 None",
    )
    core_skills: list[str] = Field(
        description="候选人的核心技能列表，只提取原文中明确出现的技能"
    )


def create_model() -> ChatOpenAI:
    return ChatOpenAI(
        model=getenv("OPENAI_MODEL", "gpt-5-nano"),
        temperature=0,
    )


def build_resume_summary_extractor(model: ChatOpenAI):
    prompt = ChatPromptTemplate.from_template(
        "请从下面简历文本中抽取候选人姓名、工作年限和核心技能。"
        "不要编造原文没有出现的信息。\n\n"
        "简历文本：\n{resume_text}"
    )

    return prompt | model.with_structured_output(ResumeSummary)


def main() -> None:
    load_dotenv()

    model = create_model()
    extractor = build_resume_summary_extractor(model)

    inputs = [
        {"resume_text": "李明，5 年 Python 后端经验，熟悉 FastAPI、Redis。"},
        {"resume_text": "王芳，3 年数据分析经验，熟悉 SQL、Python、Tableau。"},
        {"resume_text": "陈伟，前端工程师，熟悉 Vue、TypeScript、Vite。"},
    ]

    results = extractor.batch(inputs, config={"max_concurrency": 2})

    for result in results:
        print(result)


if __name__ == "__main__":
    main()
```

运行：

```powershell
uv run examples/lesson_11_batch_resume_extraction.py
```

这里没有引入新的链式语法。重点是同一条抽取链可以复用到多个输入上。

请观察：

- 每个输入都是 `{"resume_text": ...}`。
- 每个输出都是 `ResumeSummary` 对象。
- `config={"max_concurrency": 2}` 用来限制并发数量，避免一次性发起太多请求。

## 十二、抽取结果如何进入程序

拿到结构化对象后，不要只停留在打印。

你可以像普通 Python 对象一样使用它：

```python
if result.years_of_experience is not None and result.years_of_experience >= 3:
    print("进入下一轮筛选")
```

也可以转换成字典：

```python
data = result.model_dump()
```

`model_dump()` 是 Pydantic 模型的方法，用来把模型对象转换成普通 Python 字典。它适合后续写入 JSON 文件、数据库或 API 响应。

例如：

```python
{
    "candidate_name": result.candidate_name,
    "core_skills": result.core_skills,
}
```

信息抽取器的最终价值，就是让自然语言内容进入程序流程。

## 十三、常见错误与排查

### 1. 让模型抽取原文没有的信息

如果 Prompt 只写“请整理简历信息”，模型可能会补全或推测。

建议明确写：

```text
只根据原文中明确出现的信息填写字段。
不要编造原文没有出现的信息。
```

### 2. 日期格式不统一

如果 schema 描述里没有说明日期格式，模型可能输出：

```text
2020 年 7 月
2020-07
2020/07
```

建议字段描述和 Prompt 都写清楚格式，例如 `YYYY-MM`。

### 3. 电话号码用数字类型

电话号码建议用字符串。它不是数学数字，格式信息可能有业务意义。

### 4. 列表字段混入解释性文字

如果 `core_skills` 里出现“候选人熟悉 Python，因此适合岗位”这类句子，说明字段描述不够明确。

可以改成：

```python
description="候选人的核心技能列表，只包含技能词，不要包含解释性句子"
```

### 5. 嵌套对象太复杂

如果项目经历抽取经常不稳定，先减少字段，只保留项目名、角色、技术栈。

等基础抽取稳定后，再逐步增加时间、成果、业务背景等字段。

### 6. 把抽取和分类混在一起

抽取负责“原文里有什么”；分类负责“它属于什么类型、应该走哪个分支”。

本课可以有少量简单判断，但复杂分类、打标和路由会在下一课展开。

## 十四、本课小结

这一课你完成了一个简历信息抽取器。

你需要记住：

- 信息抽取器把自然语言文本转换成固定字段。
- 字段描述要明确是否允许推断、缺失时如何处理。
- 联系方式通常用字符串，而不是数字。
- 日期字段要统一格式，例如 `YYYY-MM`。
- 列表字段只放同一种元素。
- 嵌套 Pydantic 模型适合项目经历、教育经历这类多条记录。
- `prompt | structured_model` 是常见的可复用抽取链形态。
- 抽取结果可以继续用于筛选、批处理、数据库写入或后续分类。

## 十五、课后练习

### 练习 1：抽取联系方式

创建：

```text
examples/lesson_11_contact_extractor.py
```

要求：

- 定义 `ContactInfo`。
- 字段包括 `name`、`email`、`phone`、`location`。
- 可能缺失的字段使用 `str | None`。
- 明确说明只提取原文中出现的信息。

### 练习 2：抽取教育经历

创建：

```text
examples/lesson_11_education_extractor.py
```

要求：

- 定义 `EducationExperience`。
- 字段包括 `school_name`、`degree`、`major`、`graduation_date`。
- `graduation_date` 统一使用 `YYYY-MM`。
- 用 `list[EducationExperience]` 表示多段教育经历。

### 练习 3：简历批量抽取

基于本课批量抽取示例：

- 准备 5 段简历文本。
- 使用 `batch()` 批量抽取。
- 设置 `config={"max_concurrency": 2}`。
- 打印每位候选人的姓名和技能列表。

### 练习 4：把结果转成字典

在任意抽取示例中：

- 调用 `result.model_dump()`。
- 打印字典。
- 思考这个字典如何写入数据库或返回给前端。

### 练习 5：减少 schema 复杂度

设计一个过度复杂的简历抽取 schema，然后删减到：

- 8 个以内顶层字段。
- 项目经历最多 3 个子字段。
- 联系方式和日期字段都有缺失处理。

说明你删掉了哪些字段，为什么。
